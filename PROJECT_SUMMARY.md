## 📝 使用说明

### 首次使用

1. **设置个人信息**
   - 进入"设置"页面
   - 填写性别、年龄、身高、体重
   - 选择活动水平
   - 设置目标体重和周期

2. **配置 AI 估算（可选）**
   - 在"设置"页面配置大模型 API
   - 选择估算模式（规则/大模型/混合）
   - 测试连接

3. **开始记录**
   - 在"记录"页面使用 AI 描述或快捷搜索
   - 记录饮食和运动
   - 查看今日缺口状态

4. **查看统计**
   - 在"统计"页面查看趋势
   - 分析三餐分布
   - 追踪达成率

### 日常使用

- **饮食记录**: 使用 AI 描述快速记录，或搜索食物库精确记录
- **运动记录**: 选择预设运动或自定义，输入时长自动计算消耗
- **查看状态**: 今日概览实时显示缺口状态
- **追踪趋势**: 统计页面查看七日趋势和达成率

## 🎨 设计理念

### 核心原则
1. **简单优先**: 界面简洁，操作直观
2. **离线可用**: 核心功能无需网络
3. **智能估算**: AI 辅助快速记录
4. **数据驱动**: 实时计算，即时反馈
5. **隐私保护**: 所有数据本地存储

### 颜色系统
- **主色**: `#10B981` (翠绿) - 健康、活力
- **警告**: `#F59E0B` (琥珀) - 接近上限
- **危险**: `#EF4444` (红色) - 已超支
- **信息**: `#0891B2` (青色) - 运动消耗
- **体重**: `#8B5CF6` (紫色) - 体重追踪

### 交互设计
- **即时反馈**: 输入后立即计算和展示
- **颜色标识**: 绿/黄/红三色状态标识
- **响应式**: 适配桌面、平板、手机
- **容错设计**: 友好的错误提示和空状态

## 🐛 已知限制

1. **存储容量**: LocalStorage 限制 5MB，长期大量使用可能需要注意
2. **跨设备同步**: 不支持跨设备数据同步（单机版）
3. **AI 准确性**: 规则引擎估算可能不够精确，建议配置大模型 API
4. **食物库**: 内置食物库有限（28 种），需要用户自定义补充
